Module ​

Mjolnir.Mjolnir Module
julia
Mjolnir

A source-level MATLAB → idiomatic-Julia converter, written in Julia.

Pipeline (built incrementally): Parse → Resolve → Lower → Idiomatic passes → (optional, gated LLM refine) → Emit → Package assembly.

This release implements Stage 1 — Parse: a tree-sitter front-end (C FFI, no Python) that turns MATLAB source into a concrete syntax tree ([MatlabCST]).

Run deps/build.jl once before using (compiles the MATLAB grammar):

julia
julia> import Mjolnir
julia> include(joinpath(pkgdir(Mjolnir), "deps", "build.jl"))

Mjolnir.convert_project Function
julia
convert_project(srcdir, outdir; name) -> pkgdir

Convert a directory of MATLAB sources into a Julia package under outdir/name:

  • each top-level .m file -> a converted src/<file>.jl, included by the main module;

  • each MATLAB +pkg/ folder -> a Julia submodule src/Pkg.jl (capitalized) that includes its converted members;

  • each compiled MEX (foo.mexa64/.mexw64/.mexmaci64/…) has no .m source to transpile, so the binary is copied in and a binding is generated that loads + calls it via Unmex.jl (open_mex/call), with Unmex wired as a dependency;

  • Project.toml is generated by Pkg (real UUID).

Returns the package directory. Dependency wiring (adding e.g. LinearAlgebra to the generated Project.toml when converted code needs it) is not yet automated.

Diagnostics & bug reports ​

Mjolnir.conversion_report Function
julia
conversion_report(src::AbstractString) -> Dict

Build an IP-free report of everything that went wrong converting MATLAB src, suitable for filing a Mjolnir bug without sharing the source. Conversion is resilient (it does not stop at the first error), so one report captures every problem.

The report anonymizes user variable/function names to idN placeholders, strips string and number literals, drops comments, and scrubs the same names out of exception messages and todos. MATLAB keywords and recognized builtins are kept verbatim (they are MATLAB's public API). Keys:

  • summary — counts (statements, parse error, lowering exceptions, todos, placeholders)

  • problems — parse errors and caught lowering exceptions, each with an anonymized context

  • todos — scrubbed conversion todos (unhandled nodes, unmapped calls, …)

  • skeleton — the whole unit as an anonymized node-kind s-expression

See also conversion_report_json.

source
Mjolnir.conversion_report_json Function
julia
conversion_report_json(src::AbstractString) -> String

conversion_report serialized to JSON (IP-free) for sharing.

source
Mjolnir.replay_report Function
julia
replay_report(report::AbstractDict) -> Dict

Reconstruct a synthetic, IP-free MATLAB reproducer from a conversion_report and re-run the conversion on it, returning a fresh report. This lets a maintainer reproduce a reported problem locally without the original source — identifiers are placeholders, literals are dummies, only node structure and MATLAB keywords/builtins are preserved. The reconstructed source is in report["reproducer"]; replay_report runs conversion_report on it.

Best-effort: the reproducer covers the common node kinds and may not re-trigger issues that depend on specific literal values or rare constructs.

LLM refinement ​

Mjolnir.LLMBackend Type
julia
LLMBackend

Abstract supertype for optional LLM refinement backends. Concrete subtypes (HTTPBackend, SubprocessBackend, ManualBackend, FunctionBackend) implement refine. Refinement is always gated: use gated_refine, which keeps a candidate only if verify_equivalent proves it matches the deterministic baseline.

source
Mjolnir.FunctionBackend Type

In-process backend wrapping f(prompt)::String — used for testing and custom integrations.

source
Mjolnir.ManualBackend Type

Manual round-trip backend — the path for Claude via Copilot in VSCode (not scriptable): writes the prompt to promptfile, expects the model response in responsefile.

source
Mjolnir.SubprocessBackend Type

Subprocess backend for a local CLI (e.g. llama.cpp / a Tonalli runner). cmd receives the prompt on stdin and writes the completion to stdout.

source
Mjolnir.HTTPBackend Type

HTTP backend for an Anthropic- or OpenAI-compatible chat endpoint. Covers Claude directly and a local Ollama server (OpenAI-compatible). See [claude_backend] / [ollama_backend].

Mjolnir.gated_refine Function
julia
gated_refine(backend, baseline; probes, kwargs...) -> (; code, accepted)

Refine baseline with backend, then accept the candidate only if it is behaviorally equivalent to baseline on probes; otherwise return baseline unchanged. Any backend/parse error falls back to the baseline. This is the guarantee: refinement never changes behavior.

source
Mjolnir.verify_equivalent Function
julia
verify_equivalent(baseline, candidate, probes; atol, rtol) -> Bool

True if candidate reproduces baseline on every probe expression (e.g. "f(2)", "f(3.5)") evaluated in a fresh module. A candidate that errors or disagrees is not equivalent.
